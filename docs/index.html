機能

🌐

分散WireMock管理

複数のWireMockインスタンスにスタブを一括デプロイ。ワンクリックで全インスタンスを同期できます。

📈

負荷試験に最適化

複数のWireMockインスタンスを並列稼働させ、大規模な負荷試験環境を簡単に構築。

💿

SQLite永続化

外部データベース不要。シンプルなファイルベースの永続化でバックアップも簡単。

🗣

日本語UI

日本語・英語の切り替えに対応。シンプルで直感的なインターフェースでスタブを簡単に作成・管理。

🐳

Dockerサポート

Dockerイメージとして配布。docker compose up で簡単に起動できます。

📚

プロジェクト管理

開発・ステージング・本番など、環境ごとにスタブをプロジェクトとして管理。

使い方

1 プロジェクトとインスタンスを登録

プロジェクトを作成し、管理したいWireMockインスタンスを追加します。プロジェクトは環境（開発・ステージング・本番など）ごとに作成できます。各インスタンスのヘルスチェックで接続状態を確認できます。

2 スタブマッピングを作成

モックしたいAPIのリクエスト条件とレスポンスを定義します。URLパターン、HTTPメソッド、ヘッダー、ボディなど詳細な条件を設定できます。作成したスタブはSQLiteに保存され、チームで共有できます。

3 全インスタンスに同期

「全インスタンスに同期」ボタンで、作成したスタブを登録済みの全WireMockインスタンスに一括デプロイします。同期前にインスタンスをリセットするため、SQLiteとWireMockの状態が常に一致します。モックを使用後に記録されたリクエストログを画面で確認できます。
